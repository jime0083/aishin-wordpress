manual-work.txt — 人間の操作が必要な作業一覧
====================================================================
運用ルール:
- ステータス: [ ] 未着手 / [x] 完了（完了時に日付を追記） / [-] 不要になった作業
- 各Workに「期限（最悪）」を記載: progressのどのPhase/どの作業までに完了が必要か
- 各作業の詳細手順は docs/aws-deploy-guide.md（Phase 7.1 で作成）に記載予定。
  ガイド完成前に着手する場合はAIに質問すればその場で手順を案内します。
- AWS関連は「本番に限りなく近いセキュリティ・予算最小」の方針
  （Lightsail $5/月 + 格安ドメイン + 無料SSL）で構成しています。

====================================================================
Work 1: ローカル開発環境の準備
====================================================================

[ ] Work 1.1 Docker Desktop のインストールと起動
    期限（最悪）: Phase 6.1 開始まで
    ※Phase 2〜5・7はコード実装のみのため先行可能。ただしローカル動作確認が
      全てDocker待ちになるため、早く導入するほど問題の発見が早くなる。
    内容: https://www.docker.com/products/docker-desktop/ から Mac (Apple Silicon
          または Intel、お使いのMacに合わせて) 版をダウンロードしてインストールし、
          起動しておく。
    目安時間: 15分
    確認方法: ターミナルで docker --version が表示される。
    備考: Phase 1.1（ローカルWordPress構築）の前提。インストール済みなら完了に
          してください。

[-] Work 1.2 ローカルWordPressの初期インストール操作 → AIが代行（作業不要）
    期限（最悪）: —（Docker導入後にAIがWP-CLIで自動実行します。README.md参照）
    内容: AIが docker compose 環境を用意した後、http://localhost:8000 を開き
          WordPressの初期設定（言語=日本語、サイト名・管理者ユーザー名・
          パスワード・メールアドレスの入力）を行う。
          ※パスワードはパスワードマネージャー等に控えておく。
    目安時間: 5分
    確認方法: http://localhost:8000/wp-admin にログインできる。

====================================================================
Work 2: AWSアカウントの準備
====================================================================

[ ] Work 2.1 AWSアカウントの作成
    期限（最悪）: 全Phase完了後・Work 3.1 の前（progressのどのPhaseもブロックしない）
    内容: https://aws.amazon.com/jp/ から新規アカウント作成。
          メールアドレス・クレジットカード・電話番号（SMS認証）が必要。
          プランは「ベーシックサポート（無料）」を選択。
    目安時間: 15分
    費用: アカウント作成自体は無料。
    確認方法: AWSマネジメントコンソールにログインできる。

[ ] Work 2.2 ルートユーザーのMFA（二段階認証）設定
    期限（最悪）: Work 2.1 の直後（アカウント作成したら間を置かず設定する）
    内容: コンソール右上のアカウントメニュー → セキュリティ認証情報 →
          MFAデバイスの割り当て。スマホの認証アプリ（Google Authenticator等）
          を登録する。
    目安時間: 10分
    備考: 「本番同等のセキュリティ」方針の必須項目。アカウント乗っ取り対策の要。

[ ] Work 2.3 請求アラートの設定
    期限（最悪）: Work 3.1（課金が始まるインスタンス作成）の前
    内容: AWS Budgets で月額予算（例: $10）を設定し、超過見込み時にメール通知
          が来るようにする。想定外の課金を早期検知するため。
    目安時間: 10分

====================================================================
Work 3: Lightsail サーバーの作成
====================================================================

[ ] Work 3.1 Lightsail WordPressインスタンスの作成
    期限（最悪）: 全Phase完了後の公開作業時（Phase 7の手順書を見ながら実施）
    内容: Lightsailコンソール → インスタンスの作成 →
          リージョン「東京 (ap-northeast-1)」→ プラットフォーム Linux →
          設計図「WordPress (Bitnami)」→ プラン $5/月（512MB〜1GB RAM）を選択
          → 作成。
    目安時間: 10分（作成後の起動待ち含む）
    費用: 約$5/月（初回無料枠がある場合あり）
    確認方法: インスタンスが「実行中」になり、パブリックIPでサイトが開ける。

[ ] Work 3.2 静的IPの作成とアタッチ
    期限（最悪）: Work 3.1 の直後・Work 4.2（DNS設定）の前
    内容: Lightsail → ネットワーキング → 静的IPの作成 → 作成したインスタンスに
          アタッチ。再起動でIPが変わるのを防ぐ（ドメイン設定の前提）。
    目安時間: 5分
    費用: インスタンスにアタッチしている限り無料。

[ ] Work 3.3 SSH接続確認と初期パスワードの取得
    期限（最悪）: Work 5.1（本番反映）の前
    内容: Lightsailのブラウザ内SSH（またはローカルターミナル）で接続し、
          cat ~/bitnami_application_password でWordPress管理者（user）の
          初期パスワードを取得して控える。
    目安時間: 10分
    確認方法: http://<静的IP>/wp-admin に user + 取得パスワードでログインできる。

====================================================================
Work 4: ドメインとHTTPS（本番同等セキュリティ・予算最小）
====================================================================

[ ] Work 4.1 格安ドメインの取得
    期限（最悪）: Work 4.2 の前（全Phase完了後で可。取得したらドメイン名をAIに共有）
    内容: レジストラでドメインを1つ取得する。予算最小の推奨:
          - Cloudflare Registrar（原価販売・.com 約$10/年、更新も同額）
          - お名前.com / Xserverドメイン（初年度1円〜キャンペーンあり。
            ただし更新料と自動更新設定に注意）
          ポートフォリオ用なら .com / .net / .dev 等お好みで可。
          取得したドメイン名をAIに共有してください。
    目安時間: 15分
    費用: 年額 約1円〜1,500円程度（選択による）

[ ] Work 4.2 DNSレコードの設定
    期限（最悪）: Work 4.3（HTTPS化）の前
    内容: 取得したレジストラのDNS管理画面で、Aレコードを作成し
          Work 3.2 の静的IPに向ける（@ と www の2件）。
          ※Route 53は使わない（月$0.50の節約。レジストラDNSで十分）。
    目安時間: 10分（反映待ちは最大数時間）
    確認方法: http://<ドメイン>/ でサイトが表示される。

[ ] Work 4.3 HTTPS化（Let's Encrypt / bncert-tool）
    期限（最悪）: Work 5.4（本番動作確認）の前
    内容: SSHで接続し、AIが手順書に記載するコマンド
          （sudo /opt/bitnami/bncert-tool）を実行。対話プロンプトで
          ドメイン名を入力し、HTTP→HTTPSリダイレクトを有効化する。
          証明書は自動更新される。
    目安時間: 15分
    費用: 無料
    確認方法: https://<ドメイン>/ が鍵マーク付きで表示され、httpアクセスが
              httpsへリダイレクトされる。

Work 5: 本番デプロイと初期設定
====================================================================

[ ] Work 5.1 テーマ・プラグインの本番反映
    期限（最悪）: 公開したいタイミング（全Phase完了・Work 3完了後）
    内容: docs/deploy-package.md（Phase 7.2で作成）に従い、
          (1) 管理画面からテーマzipをアップロード・有効化
          (2) ACFプラグインをインストール・有効化
          (3) SSH上でシードスクリプトを実行（固定ページ・インタビュー・画像投入）
    目安時間: 30分
    確認方法: 本番URLで全ページが表示される。

[ ] Work 5.2 WordPressセキュリティ初期設定
    期限（最悪）: Work 5.4（本番動作確認）の前。特に管理者パスワード変更は
    サイト公開（DNS向け先変更）と同時に必ず実施
    内容: (1) 管理者パスワードを強固なものに変更（初期userのままにしない）
          (2) 不要なデフォルトプラグイン・テーマの削除
          (3) ログイン試行制限プラグイン（Limit Login Attempts Reloaded）の
              インストール・有効化
          (4) 設定 > 一般 のサイトURLがhttpsになっていることを確認
          (5) WordPress本体・プラグインの自動更新を有効化
    目安時間: 20分

[ ] Work 5.3 Lightsail自動スナップショット（バックアップ）の有効化
    期限（最悪）: 本番公開から1週間以内
    内容: Lightsail → インスタンス → スナップショット → 自動スナップショットを
          有効化（日次・7世代保持）。
    目安時間: 5分
    費用: スナップショット容量分の従量課金（月$0.5前後の見込み）

[ ] Work 5.4 本番動作確認
    期限（最悪）: 本番公開（DNS切替・HTTPS化）の直後
    内容: docs/production-checklist.md（Phase 7.3で作成）に従い、
          全ページの表示・アニメーション・フォーム・スマホ実機表示・
          証明書の有効性を確認する。問題があればAIに共有
          （problem.txt に記録して修正します）。
    目安時間: 30分
